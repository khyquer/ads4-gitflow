
# GitFlow Lab
Repositório para praticar Git Flow (feature, release, hotfix).

## Instruções iniciais
1. Clone o repositório:
   git clone https://github.com/khyquer/ads4-gitflow.git
2. Entre na pasta:
   cd ads4-gitflow
3. Crie a sua branch:
   git checkout -b feature/<nome-aluno>


A Gerência de Configuração de Software (GCS) é a disciplina que controla as modificações em um sistema. Ela é crucial para reduzir custos de manutenção, que podem chegar a 75% do total do projeto, e para garantir a produtividade e a rastreabilidade do processo.

A GCS se baseia em três pilares principais:

Controle de Versões: Gerencia a evolução dos artefatos do projeto, evitando perdas de trabalho e conflitos entre desenvolvedores. Usa conceitos como branches (linhas de desenvolvimento isoladas) e merge (combinação de branches). O Versionamento Semântico (SemVer) é usado para padronizar a comunicação sobre compatibilidade de versões (MAJOR.MINOR.PATCH).

Controle de Modificações: Rastreia e avalia as solicitações de mudança (como correções de bugs ou novas funcionalidades).

Gerenciamento de Construção: Automatiza o processo de transformação do código em um sistema executável.
